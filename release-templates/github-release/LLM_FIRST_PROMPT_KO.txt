아래 GitHub Release 패키지를 npm exec로 실행해서 Hare M365 Agent를 사용해.

패키지 URL:
__PACKAGE_URL__

중요: Claude/Cowork처럼 도메인 허용 목록이 있는 환경에서는 npm 실행 전에 먼저 사용자에게 아래 도메인이 허용되어 있는지 확인시켜.

- github.com
- release-assets.githubusercontent.com
- registry.npmjs.org
- graph.microsoft.com
- login.microsoftonline.com

사용자가 "도메인 허용 완료"라고 확인하기 전에는 npm exec, npx, curl, 다운로드, 로그인, Outlook/Teams/Files 조회를 시작하지 마.

프로젝트 또는 배포 폴더가 연결되어 있으면 같은 연결 폴더 안의 ./runtime을 사용해. 모든 Hare 명령에 HARE_M365_DATA_DIR=./runtime을 붙여 실행해.

도메인이 아직 허용되지 않았다면 사용자에게 이렇게 안내해.

1. Claude/Cowork 설정을 연다.
2. 기능 또는 네트워크 액세스 설정으로 이동한다.
3. 도메인 허용 목록에 위 5개 도메인을 추가한다.
4. 저장 후 이 채팅으로 돌아와 "도메인 허용 완료"라고 말한다.

사용자가 도메인 허용을 완료했다고 말하면 doctor/auth status로 설정과 로그인을 확인한 뒤 Microsoft 365 요청을 처리해.

도메인 허용 완료 전에는 폴더 연결 요청도 시작하지 마. 도메인 허용 후 먼저 현재 실행환경에서 doctor/auth status를 시도해.

loggedIn이 false이면 로그인 hard gate로 멈추고 Outlook/Teams/Files 조회를 실행하지 마.

loggedIn이 false일 때 폴더 연결 요청이나 지속 인증 캐시 위치 확보로 우회하지 마. 사용자가 명시적으로 "연결 폴더로 진행"이라고 요청한 경우에만 Claude/Cowork 폴더 연결을 요청해.

Claude Cowork나 hosted sandbox처럼 명령 출력이 주로 LLM에게 보이는 환경에서는 auth login을 자동 실행하지 마. 사용자가 직접 볼 수 있는 로컬 터미널에서 실행할 로그인 명령을 안내하고 멈춰. 연결 폴더가 이미 있는 경우에만 사용자가 같은 연결 폴더에서 같은 HARE_M365_DATA_DIR=./runtime 설정으로 로그인해야 해.

사용자가 명시적으로 "로그인 시작"을 요청했고, 현재 터미널/브라우저 출력이 사용자에게 직접 보이는 환경에서만 auth login을 1회 실행할 수 있어. 실패하면 반복 재시도나 도메인 진단 루프를 돌지 말고 오류를 보고하고 멈춰.

Microsoft device code는 사용자가 화면에서 직접 보고 브라우저에 입력하게 해. 사용자가 로그인 완료 후 "로그인 완료"라고 알려주면 doctor/auth status를 다시 확인한 뒤 원래 요청을 이어서 처리해.

패키지 다운로드가 403, network_error, proxy, allowlist 오류로 실패하면 Windows GUI나 파일 탐색기로 우회하지 말고, 어떤 도메인이 막혔는지 사용자에게 알려줘.

실행 예:
HARE_M365_DATA_DIR=./runtime npm exec --yes --package "__PACKAGE_URL__" -- hare-m365 doctor
HARE_M365_DATA_DIR=./runtime npm exec --yes --package "__PACKAGE_URL__" -- hare-m365 auth status
