Hare M365 Agent 시작 가이드

LLM이 Outlook, Teams, OneDrive, SharePoint를 읽기 전용으로 조회하기 위한 사전 설정 안내입니다.

먼저 해야 할 일: Claude/Cowork에서는 npm 실행 전에 도메인 허용 목록을 먼저 설정해야 합니다. 이 단계를 건너뛰면 LLM이 패키지를 다운로드하기도 전에 실패합니다.

2폴더 허용 요청이 나오면

LLM이 먼저 Hare M365 Agent 전용 폴더 연결을 요청하도록 프롬프트에 포함되어 있습니다. 사용자는 폴더 허용 또는 선택 창이 뜨면 아래 전용 폴더만 승인합니다.

Windows: 문서 > Hare M365 Agent
Mac: Documents > Hare M365 Agent
  1. Claude/Cowork가 Hare M365 Agent 폴더 연결을 요청하면 허용 또는 선택을 누릅니다.
  2. 폴더 선택창이 직접 열리면 문서 또는 Documents 안의 Hare M365 Agent 폴더를 선택합니다.
  3. 해당 폴더가 없으면 새 폴더를 만들고 이름을 Hare M365 Agent로 입력한 뒤 선택합니다.

SharePoint 공용 폴더, Teams 공유 폴더, 다운로드 폴더 전체, 바탕화면 전체는 선택하지 마세요.

3맨 처음 1회용 프롬프트 전달

설치 직후 처음 사용하는 채팅, 또는 도메인 허용 여부가 확실하지 않은 채팅에서는 아래 프롬프트를 LLM에게 한 번 전달합니다.

아래 GitHub Release 패키지를 npm exec로 실행해서 Hare M365 Agent를 사용해.

패키지 URL:
__PACKAGE_URL__

중요: Claude/Cowork처럼 도메인 허용 목록이 있는 환경에서는 npm 실행 전에 먼저 사용자에게 아래 도메인이 허용되어 있는지 확인시켜.

- github.com
- release-assets.githubusercontent.com
- registry.npmjs.org
- graph.microsoft.com
- login.microsoftonline.com

사용자가 "도메인 허용 완료"라고 확인하기 전에는 npm exec, npx, curl, 다운로드, 로그인, Outlook/Teams/Files 조회를 시작하지 마.

폴더 접근 권한이 없으면 파일 탐색기나 데스크톱 제어로 우회하지 말고, Claude/Cowork의 폴더 연결 기능으로 Hare M365 Agent 전용 폴더 연결/허용을 먼저 요청해. 요청 메시지에는 "문서/Documents 안의 Hare M365 Agent 폴더"라고 명확히 안내해. 사용자가 임의 경로를 판단하게 하지 말고, SharePoint 공용 폴더, Teams 공유 폴더, 다운로드 폴더 전체, 바탕화면 전체는 선택하게 하지 마.

프로젝트 또는 배포 폴더가 연결되어 있으면 같은 연결 폴더 안의 ./runtime을 인증 캐시 위치로 사용해. 모든 Hare 명령에 HARE_M365_DATA_DIR=./runtime을 붙여 실행해.

CLI는 runtime/.cache/msal-cache.json을 사용해 Microsoft Graph 인증을 수행할 수 있어. 하지만 너는 runtime/.cache/msal-cache.json 파일을 열거나 출력하거나 요약하거나 업로드하지 마.

사용자가 도메인 허용을 완료했다고 말하면 llm-guide를 읽고 doctor/auth status로 설정과 로그인을 확인한 뒤 Microsoft 365 요청을 처리해.

loggedIn이 false이면 로그인 hard gate로 멈추고 Outlook/Teams/Files 조회를 실행하지 마.

Claude Cowork나 hosted sandbox처럼 명령 출력이 주로 LLM에게 보이는 환경에서는 auth login을 자동 실행하지 마. 사용자가 직접 볼 수 있는 로컬 터미널에서 실행할 로그인 명령을 안내하고 멈춰. 연결 폴더를 쓰는 경우 사용자가 같은 연결 폴더에서 같은 HARE_M365_DATA_DIR=./runtime 설정으로 로그인해야 해.

사용자가 명시적으로 "로그인 시작"을 요청했고, 현재 터미널/브라우저 출력이 사용자에게 직접 보이는 환경에서만 auth login을 1회 실행할 수 있어. 실패하면 반복 재시도나 도메인 진단 루프를 돌지 말고 오류를 보고하고 멈춰.

Microsoft device code는 사용자가 화면에서 직접 보고 브라우저에 입력하게 해. 코드를 채팅에 복사하거나 반복하지 마. 사용자가 로그인 완료 후 "로그인 완료"라고 알려주면 doctor/auth status를 다시 확인한 뒤 원래 요청을 이어서 처리해.

.env, .cache, runtime/.cache, token, device code는 읽거나 출력하지 마.
4로그인 필요 시 사용자가 직접 완료

loggedIn: false이면 LLM은 로그인 hard gate에서 멈추고, 사용자가 직접 볼 수 있는 로컬 터미널에서 실행할 로그인 명령을 안내합니다. 사용자는 LLM이 안내한 로그인 명령을 그대로 실행하면 됩니다.

  1. 사용자가 로컬 터미널에서 로그인 명령을 실행하면 검은 명령창 또는 터미널에 Microsoft 로그인 안내가 표시됩니다.
  2. 문장 끝에 보이는 짧은 코드가 device code입니다. 아래 예시의 빨간 박스 위치입니다.
  3. 브라우저의 Microsoft device login 화면에 그 코드를 사용자가 직접 입력합니다.
  4. 브라우저가 자동으로 열리지 않으면 터미널의 https://microsoft.com/devicelogin 링크를 Ctrl+클릭합니다. Mac에서는 Command+클릭을 시도하거나 주소를 브라우저 주소창에 직접 입력합니다.
  5. 회사 Microsoft 계정으로 로그인한 뒤, LLM에게 로그인 완료라고 알려줍니다.
명령창에 표시되는 로그인 코드 예시

도메인 허용은 이미 완료했다. Hare M365 Agent를 사용해줘.

패키지 URL:
__PACKAGE_URL__

6직접 테스트 명령

터미널에서 직접 확인해야 할 때는 아래 명령을 사용합니다.

npm exec --yes --package "__PACKAGE_URL__" -- hare-m365 llm-guide
npm exec --yes --package "__PACKAGE_URL__" -- hare-m365 doctor
npm exec --yes --package "__PACKAGE_URL__" -- hare-m365 auth status
